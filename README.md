# resume-intake-workflow

用于通过飞书 Agent 将候选人简历自动录入到多维表格。

## 业务目标
将候选人简历（PDF）自动录入到多维表格「**招聘进度管理 - 2025年应聘人员登记**」中，包括：
- 基本信息字段
- 原始 PDF 附件

## 前提条件
- 飞书账号已接入 Agent
- 发送简历前，确认目标多维表格已给 Agent 开启编辑权限

## 使用方式

### 方式一，单份简历
**适用场景：** 候选人单独发送一份 PDF 简历

**操作步骤：**
1. 在飞书与 Agent 的私聊窗口，直接发送 PDF 文件
2. 等待 Agent 自动完成处理
3. Agent 回复“已录入成功”即完成

**示例：**
> 你发送 `张三_简历.pdf`
>
> Agent 回复：`张明，已录入成功，字段+附件均已写入`

### 方式二，批量简历（ZIP 包）
**适用场景：** 一次性收集了多份简历，需要批量录入

**操作步骤：**
1. 将多份 PDF 简历打包为 ZIP 文件
2. ZIP 内只放 PDF，不要混入其他文件
3. 在 Agent 私聊窗口，直接发送 ZIP 文件
4. 等待 Agent 自动处理
5. Agent 回复最终汇总，例如：`已处理 8 份，成功 8 份`

**示例：**
> 你发送 `4月简历汇总.zip`（内含 8 个 PDF）
>
> Agent 回复：`已处理 8 份，成功 8 份`

## 录入结果说明

| 结果类型 | 含义 | 是否需要处理 |
| --- | --- | --- |
| 成功 | 字段 + 附件全部写入 | 无需处理 |
| 部分成功 | 字段写入成功，附件失败 | 联系技术人员排查 |
| 失败 | 字段写入失败 | 重新发送，或联系技术人员排查 |

## 常见问题

### Q1：可以发送 Word、Excel 吗？
不可以，目前只支持 PDF 格式。

### Q2：简历里没有姓名怎么办？
Agent 会自动跳过该简历，并在汇总中标注失败。不允许创建空白记录，请确认候选人姓名后再重新发送。

### Q3：发送后没有响应？
请检查 PDF 文件是否损坏，或重新发送一次。如果持续无响应，联系技术人员。

### Q4：可以只分析简历不录入吗？
可以，在发送时注明“只分析，不录入”，Agent 会按要求仅做内容分析。

### Q5：多维表格在哪里看？
表格名称为「招聘进度管理 - 2025年应聘人员登记」，在飞书多维表格中搜索即可找到。

## 安全护栏说明
以下情况 Agent 会自动拒绝处理：
- 缺少候选人姓名，不会创建空白记录
- 非 PDF 格式文件，不会录入
- 目标表格字段与简历不匹配时，按预设规则保守处理，不擅自添加字段
