# 简历录入字段映射

## 目录

- 已批准的安全字段
- 提取原则
- 当前启发式规则
- 输出格式

## 已批准的安全字段

固定生产链路里，只填写这些可靠度足够高的字段：

- 应聘者姓名
- 年龄
- 应聘岗位
- 联系方式
- 学历
- 毕业院校
- 专业
- 是否为全日制
- 最近一家公司名称
- 目前薪资
- 期望薪资
- 附件

除非业务规则被明确扩展，否则其他字段一律留空。

## 提取原则

- 不得编造。
- 宁可留空，也不要填错。
- 保持保守归一化。
- 如果薪资文本有歧义、不是数字、或包含 `面议` / `保密` / `详谈` 等字样，薪资字段留空。

## 当前启发式规则

### 姓名

优先匹配明确的 `姓名:` 模式。

为降低 PDF OCR / 字符间距问题带来的漏识别，需要先做稳健归一化，再按以下顺序提取：
- `姓名:` / `姓名：` / 带空格断裂的同类模式
- 简历前几行中的独立姓名片段
- PDF 文件名中的可靠兜底姓名

候选人姓名只能来自简历正文或 PDF 文件名，不能来自消息发送者姓名。

### 联系方式

若能提取到手机号和邮箱，则组合成 `phone / email`；只命中一个时，只返回那一个。

### 年龄

按以下顺序提取：
- 明确的 `年龄:` / `年龄：` 模式
- 明确的 `xx岁` 模式
- 明确带标签的 `出生年月` / `出生日期` / `生日`，允许推导年龄

没有明确标签或推导依据时，年龄留空。

### 学历

当前关键词扫描顺序：

- 博士
- 硕士
- 本科
- 大专
- 中专
- 高中

### 毕业院校

使用以 `大学` 或 `学院` 结尾的保守匹配模式。

### 专业

优先匹配明确的 `专业:` 模式。

### 是否全日制

映射规则：

- `全日制` => `全日制`
- `非全日制|成人教育|自考|函授` => `非全日制`

### 最近一家公司名称

优先匹配 `最近一家公司` / `最近公司` / `现公司` / `就职于` 等显式模式。

### 应聘岗位

优先匹配 `应聘岗位` / `求职意向` / `意向岗位` 等显式模式。

必要时，需要从同一行中剔除后续无关标签，例如：

- 意向城市
- 期望薪资
- 电话
- 邮箱
- 性别
- 年龄
- 现所在地
- 最高学历

## 输出格式

只返回有值的字段，不输出空字符串字段。

示例：

```json
{
  "应聘者姓名": "王楠",
  "联系方式": "13800000000 / test@example.com",
  "学历": "本科"
}
```
